# Comm Design Journal 2026-06-05

| Time | Area | Entry | Follow-up |
|---|---|---|---|
| 2026-06-05 | Boundary | Defined Data Bus as durable bundle-scoped inbound messaging, separate from conversation ingress and the transient comm relay. | Implement runtime package after docs review. |
| 2026-06-05 | Transport | Agreed the browser can use the same authenticated Socket.IO connection, but the package is `data_bus.publish`, not `chat_message`. | Add Socket.IO ingress handler that routes to Data Bus Redis Streams. |
| 2026-06-05 | Ordering | Clarified that Redis consumer groups do not serialize same-object work by themselves. `serial_per_partition` must be guaranteed by SDK runtime locks, and storage still enforces idempotency and `base_revision`. | Implement Redis token locks and conflict tests. |
| 2026-06-05 | ReAct bridge | Data Bus messages do not create `ev:` or timeline records by default. A bundle must explicitly bridge handled state into conversation ingress when an agent should see it. | Keep task-tracker canvas state traffic separate from focused chat context. |
| 2026-06-05 | Docs | Added [Data Bus](../../data-bus-README.md) and expanded [Data Bus Runtime Handoff](../databus-runtime.md). | Hand off implementation slices to the runtime agent. |
| 2026-06-05 | Docs | Added the central [Conversation Event Bus And Data Bus](../../conversation-event-bus-and-data-bus-README.md) contract and linked bundle-facing client docs to it. | Keep bundle docs short and point builders to the central bus split. |
