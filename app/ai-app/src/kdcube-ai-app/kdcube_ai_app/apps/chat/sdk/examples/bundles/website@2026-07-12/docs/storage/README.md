# Website App Storage

| Data | Owner | Persistence |
| --- | --- | --- |
| Site registration and composition | `bundles.yaml` app config | Descriptor lifecycle |
| Website source | App package `ui/site` | Source control/release |
| Built website | App local UI build output | Rebuildable cache |
| Site routing projection | Redis catalog + generation | Rebuildable distribution snapshot |
| Request-time site catalog | Proc memory | Immutable hot cache |
| Platform auth config | Platform/Connection Hub | Read through `/api/cp-frontend-config` |
| User session | Platform authority | Read through `/profile`; not copied into app storage |

The app has no secrets and no canonical mutable business data. Built UI output
is disposable and is regenerated by the standard app main-view lifecycle.
The site registry itself is derived from the active runtime app registry and
descriptor authority; it is not a second persisted source of truth. Redis is
read on startup/reconnect and updated on configuration changes, never for each
site request.
